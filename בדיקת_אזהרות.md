# 🔍 בדיקת אזהרות ב-Railway

## ✅ סטטוס נוכחי:

כל ה-Services **Online**! זה מעולה! 🎉

אבל יש **אזהרות** ב-Worker ו-API Services. בואו נבדוק מה זה.

---

## 🔍 איך לבדוק את האזהרות:

### דרך Dashboard:

1. **Worker Service** → **לחץ עליו**
2. **"Metrics"** או **"Logs"** → **חפש אזהרות**
3. **API Service** → **לחץ עליו**
4. **"Metrics"** או **"Logs"** → **חפש אזהרות**

### דרך CLI:

```powershell
# בדוק Logs של Worker Service
pnpm railway:logs:worker

# בדוק Logs של API Service
pnpm railway:logs:api
```

---

## 📋 מה לבדוק:

### 1. Worker Service:

- ✅ האם `REDIS_URL` מוגדר?
- ✅ האם Worker מתחבר ל-Redis?
- ✅ האם יש שגיאות ב-Logs?

### 2. API Service:

- ✅ האם `DATABASE_URL` מוגדר?
- ✅ האם API מתחבר ל-PostgreSQL?
- ✅ האם יש שגיאות ב-Logs?

---

## 🎯 מה לעשות:

### אם האזהרות לא קריטיות:

**אין צורך לעשות כלום!** אם הכל עובד, האזהרות יכולות להיות:
- Memory usage גבוה
- CPU usage גבוה
- Warnings שלא משפיעים על הפונקציונליות

### אם האזהרות קריטיות:

1. **בדוק את ה-Logs** → ראה מה השגיאה
2. **תקן את הבעיה** → לפי השגיאה
3. **Redeploy** → אם צריך

---
