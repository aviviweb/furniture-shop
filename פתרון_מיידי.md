# ⚡ פתרון מיידי - Database Error

## 🔴 הבעיה:
```
Can't reach database server at ...:5432
```

## ✅ מה לעשות (3 דקות):

### שלב 1: בדוק אם יש PostgreSQL
1. **Railway Dashboard** → **חזור לרשימת Services**
2. **חפש "Postgres"** או "PostgreSQL"
3. **אם אין** → צור אחד (ראה שלב 2)
4. **אם יש** → עבור לשלב 3

### שלב 2: צור PostgreSQL (אם אין)
1. **לחץ "New"** → **"Database"** → **"PostgreSQL"**
2. **חכה 30 שניות**
3. **`DATABASE_URL` יתווסף אוטומטית!**

### שלב 3: וודא ש-DATABASE_URL מוגדר ב-API
1. **PostgreSQL Service** → **"Variables"** → **חפש `DATABASE_URL`** → **העתק**
2. **API Service** → **"Variables"** → **"+ New Variable"**
3. **שם:** `DATABASE_URL` → **ערך:** הדבק מה שהעתקת → **שמור**

### שלב 4: הרץ Migrations
1. **API Service** → **"Deployments"** (בתפריט משמאל)
2. **למעלה מימין** → **לחץ "Run Command"** (או "Shell")
3. **ייפתח חלון** → **הדבק:**
   ```
   pnpm --filter @furniture/prisma migrate deploy
   ```
4. **Enter** → חכה לסיום

**💡 לא מוצא?** → ראה `איפה_להריץ_MIGRATIONS_בRAILWAY.md`

**⚠️ אם אתה רואה שגיאה "localhost:5432":**
→ אתה מנסה להריץ מקומית! ראה `תיקון_MIGRATIONS_מקומי.md`

### שלב 5: Restart
1. **API Service** → **לחץ "Restart"** (כפתור אדום)
2. **חכה 2-3 דקות**
3. **בדוק Logs** - אמור לעבוד!
